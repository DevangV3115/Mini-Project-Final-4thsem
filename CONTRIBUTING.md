# Contributing to Self-Correcting Reasoning Framework

First off, thank you for considering contributing to our project. It's people like you that make this framework such a great tool.

## Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our Issues tab first. If not found, open a new issue.

## Pull Requests

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## Any contributions you make will be under the Project License

When you submit code changes, your submissions are understood to be under the same License that covers the project.

## Code formatting

- We use ESLint and Prettier for the frontend.
- We use Black for Python formatting on the backend.
- Run the linting scripts locally before pushing.
